## 1. User Intent Discovery

### Core Problem
사주(동양 운세)와 타로(서양 운세)를 실시간 배틀 쇼 형식으로 대결시키는 엔터테인먼트 웹 서비스.
같은 질문에 대해 두 가지 완전히 다른 점술 체계가 어떤 해석을 내리는지 비교하는 재미를 제공한다.

### Target Users
- 운세/점술에 관심 있는 일반 사용자
- 재미와 공유성을 중시하는 2030 세대

### Success Criteria
1. **재미 + 공유성**: 결과를 SNS에 공유하고 싶게 만드는 배틀 쇼 경험
2. **정확성 + 깊이**: 실제 사주/타로 이론에 기반한 해석 (추측 금지)

### User Input
- **필수**: 생년월일(양력) + 성별
- **선택**: 출생시간 (입력 시 시주 포함 → 더 정확한 사주 분석)

## 4. Architecture

### Tech Stack
- HTML5 / CSS3 / Vanilla JavaScript (ES6+)
- Claude API (AI 해석 생성)
- html2canvas (결과 이미지 생성)

### File Structure
```
index.html              ← 단일 HTML (5개 섹션 SPA)
css/
  style.css             ← 메인 스타일
  animations.css        ← 배틀 애니메이션 전용
js/
  app.js                ← 앱 초기화 + 화면 전환
  saju.js               ← 사주 계산 엔진
  tarot.js              ← 타로 카드 덱 + 드로우
  battle.js             ← 배틀 진행 + 투표 + 판정
  ai-interpreter.js     ← Claude API 호출
  share.js              ← 결과 이미지 + SNS 공유
data/
  cheongan.json         ← 천간 10개 (甲~癸)
  jiji.json             ← 지지 12개 (子~亥)
  sipsung.json          ← 십성 데이터
  tarot-major.json      ← 메이저 아르카나 22장
  tarot-minor.json      ← 마이너 아르카나 56장
assets/
  images/               ← VS 로고, 배경, 타로 카드
```

### Page Flow
```
[인트로] → [입력] → [배틀 3R] → [결과] → [공유]
```

## 5. Key Components

### 5.1 사주 엔진 (saju.js)
- 만세력 기반 양력→사주팔자 변환
- 일간(日干) 확정 → 오행 분포 → 십성 배치 → 신강/신약 판단
- 출생시간 미입력 시 시주 제외 (6글자 분석)
- **1번 계산 → 3라운드 모두 동일 데이터 재사용** (결정론적)

### 5.2 타로 엔진 (tarot.js)
- 78장 덱 (메이저 22 + 마이너 56)
- 라운드별 스프레드:
  - R1 연애운: 3장 (과거-현재-미래)
  - R2 재물운: 3장 (현재상황-조언-결과)
  - R3 종합운세: 5장 (파이브카드)
- 총 11장 드로우, 중복 없음 (덱에서 순차 소비)
- **라운드마다 새 카드** (확률론적)

### 5.3 배틀 엔진 (battle.js)
- 3라운드 고정: 연애운 → 재물운 → 종합운세
- 하이브리드 진행: 자동 진행 + 유저 투표 + AI 판정
- 최종 점수: 유저투표(60%) + AI종합판정(40%)

### 5.4 AI 해석기 (ai-interpreter.js)
- Claude API 호출 (라운드당 2회, 총 7회)
- 사주 프롬프트: 사주팔자 + 오행/십성 데이터 + 라운드 주제
- 타로 프롬프트: 드로우 카드 + 정/역방향 + 라운드 주제
- 해석 길이: 150~200자 (읽기 편한 분량)
- 라운드 시작 시 지연 로딩 (애니메이션으로 대기 커버)

### 5.5 공유 모듈 (share.js)
- html2canvas로 결과 카드 이미지 생성
- 카카오톡 / Twitter / URL 복사

## 6. Data Flow

```
[유저 입력]
    ↓
[사주 계산 (1회)] + [타로 드로우 (라운드별)]
    ↓
[AI 해석 생성 (라운드 시작 시)]
  R1: 사주 연애운 + 타로 연애운  (API 2회)
  R2: 사주 재물운 + 타로 재물운  (API 2회)
  R3: 사주 종합운세 + 타로 종합운세 (API 2회)
    ↓
[유저 투표 (라운드별)]
    ↓
[최종 판정 (API 1회)]
  투표(60%) + AI(40%) = 최종 승자
    ↓
[결과 카드 이미지 생성 + SNS 공유]
```

**API 호출**: 총 7회 (R1: 2 + R2: 2 + R3: 2 + Final: 1)

## 7. 사주/타로 이론 기반 데이터

### 7.1 사주 핵심 데이터
- **천간 10개**: 甲乙丙丁戊己庚辛壬癸 (각각 오행+음양 매핑)
- **지지 12개**: 子丑寅卯辰巳午未申酉戌亥 (각각 오행+음양+지장간 매핑)
- **오행 상생**: 목→화→토→금→수→목
- **오행 상극**: 목→토→수→화→금→목
- **십성 10종**: 비견, 겁재, 식신, 상관, 편재, 정재, 편관, 정관, 편인, 정인

### 7.2 타로 핵심 데이터
- **메이저 아르카나 22장**: 0 The Fool ~ 21 The World (정/역방향 키워드)
- **마이너 아르카나 56장**: 4수트(완드/컵/소드/펜타클) x 14장(A~10 + 시종/기사/여왕/왕)
- **연애 핵심 카드**: The Lovers, Two of Cups, Empress, Knight of Cups 등
- **재물 핵심 카드**: Ace of Pentacles, Nine of Pentacles, Wheel of Fortune 등

### 7.3 AI 프롬프트 전략
- 사주: "사주 전문가로서 {사주팔자}를 분석하여 {주제}에 대해 해석해주세요"
- 타로: "타로 전문가로서 {드로우 카드 목록}을 기반으로 {주제}에 대해 해석해주세요"
- 공통: 실제 이론 용어 사용, 재미있지만 정확한 톤, 150~200자

## 8. Battle Format

| Round | 주제 | 사주 분석 포인트 | 타로 스프레드 |
|-------|------|-----------------|--------------|
| R1 | 연애운 | 재성/관성, 도화살, 홍염살 | 3장 (과거-현재-미래) |
| R2 | 재물운 | 식상생재, 비겁쟁재, 용신 | 3장 (상황-조언-결과) |
| R3 | 종합운세 | 대운/세운, 용신/기신 | 5장 (파이브카드) |
| Final | 종합 판정 | 전체 사주 종합 | 전체 카드 종합 |

## 10. Risk & Mitigation

| Risk | Impact | Mitigation |
|------|--------|------------|
| 만세력 변환 복잡도 | 높음 | 만세력 데이터 테이블(JSON) 사용, 라이브러리 검토 |

| API 키 프론트엔드 노출 | 중간 | MVP 한정, 프로덕션 시 백엔드 프록시 필수 |
| 사주/타로 해석 품질 | 높음 | 실제 이론 데이터 기반 프롬프트, 이론 용어 포함 |
