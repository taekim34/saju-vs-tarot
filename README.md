# 사주 vs 타로 — 운명의 대결

> 동양의 사주(四柱)와 서양의 타로가 당신의 운세를 두고 벌이는 3라운드 AI 배틀!

## 만들게 된 계기

이 프로젝트는 [bkamp 유튜브 라이브 방송](https://www.youtube.com/watch?v=HNjQjW37mpI)에서 **바이브코딩 교육용 시연**으로 제작되었습니다. 라이브 방송 중 실시간으로 코딩을 진행하고, 방송 종료 후 완성하여 실제 배포까지 마무리한 프로젝트입니다.

"프레임워크 없이, AI와 대화하며 얼마나 빠르게 서비스를 만들 수 있는가"를 보여주기 위한 샘플이지만, 하다 보니 욕심이 나서 **실제 유료 서비스 급**까지 구현했습니다.

## 서비스 흐름

```
인트로 → 생년월일/성별 입력 → 7개 주제 중 3개 선택 → 3라운드 배틀 → 최종 결과 → 공유/통계
```

각 라운드마다 사주와 타로가 각각 AI 해석을 제시하고, 사용자 투표(60%) + AI 판정(40%)으로 승자를 결정합니다. 처음에는 연애/재물/종합으로 고정이었으나, 7개 주제(연애운, 재물운, 직업운, 건강운, 학업운, 대인관계, 종합운세) 중 3가지를 직접 선택하도록 확장했습니다.

## 사주 엔진 수준

사주 엔진(`js/saju.js`)은 단순 오락용이 아닌, **실제 전문 사주 서비스에 근접한 수준**으로 구현되어 있습니다:

| 기능 | 설명 |
|------|------|
| 사주팔자 산출 | 년주·월주·일주·시주 정확한 간지 계산 (줄리안 일수 기반) |
| 절기 정밀 계산 | 2024~2028 연도별 절기 경계 테이블 (입춘, 경칩 등 정확한 날짜) |
| 십성(十星) | 비견·겁재·식신·상관·편재·정재·편관·정관·편인·정인 — 음양 버그 수정 완료 |
| 지장간(支藏干) | 숨은 천간 추출 및 오행 가산 분석 |
| 충(冲)·합(合) | 6충 + 삼합·반합·육합 판단 및 의미 해석 |
| 대운(大運) | 10년 단위 운의 흐름 계산 (순행/역행, 현재 대운 표시) |
| 12운성 | 천간 에너지의 생로병사 순환 (장생~양) |
| 세운(歲運) | 올해 간지와 사주의 관계 분석 |
| 신강/신약 | 지장간 + 12운성 가중치 반영 정밀 판단 |
| 신살/길성 | ~30종 (도화살·홍염살·역마살·천을귀인·문창귀인 등) |
| 격국 판단 | 십성 집계 기반 격국 분류 |
| 오행 생극 관계도 | 상생·상극 시각화 |
| 인생 로드맵 | 연령대별 대운 흐름 해석 (종합운세) |
| 음력 변환 | 1900~2050 음양력 변환 (외부 API 없이 자체 테이블) |
| 지방시 보정 | -30분 보정으로 시주 정확도 향상 |

> 상용 서비스 대비 부족한 점: 절기 테이블이 2024~2028로 한정, 전문 역학자 감수 미진행. 오락 목적으로 충분하며, 테이블 확장과 감수를 거치면 상용 수준으로 확장 가능합니다.

## 타로 엔진 수준

타로 엔진(`js/tarot.js`)은 **Golden Dawn 전통을 반영한 전문가 수준** 분석을 제공합니다:

| 기능 | 설명 |
|------|------|
| 78장 풀 덱 | 메이저 아르카나 22장 + 마이너 아르카나 56장 (실제 카드 이미지 포함) |
| 역방향 35% | 실제 타로 리더 수준의 역방향 확률 |
| 3종 스프레드 | 쓰리 카드(과거-현재-미래), 쓰리 카드(상황-조언-결과), 호스슈(7장) |
| 원소 상성 | Elemental Dignity — 인접 카드 간 우호/적대/중립 관계 분석 |
| 수트·숫자 패턴 | 반복 수트의 원소 에너지 + 수비학적 메시지 감지 |
| 코트 카드 | 인물 에너지 분석 (시종·기사·여왕·왕) |
| 시그니피케이터 | 나이·성별·주제·사주 오행 기반 자동 배정 |
| 메이저/마이너 비율 | 운명 개입 강도 판단 |

## 통계 기능

배틀 결과를 다양한 관점에서 사주 vs 타로 선호도를 비교합니다:

- **성별 분석**: 남성/여성의 사주 vs 타로 선호도 비교
- **연령대별 분석**: 10대~80대 각 연령대의 선호 경향
- **주제별 투표**: 7개 주제별로 사주/타로 중 어느 쪽 해석이 더 선택받았는지
- **종합 전적**: 역대 전체 참가자의 사주 vs 타로 승률

## 기술 스택 — 의도적으로 가볍게

시연용 프로젝트이므로 **별도의 프레임워크 없이** 가장 가벼운 구성을 선택했습니다:

| 항목 | 스택 |
|------|------|
| 프론트엔드 | Vanilla JS SPA (~5,000줄), IIFE 모듈 패턴 |
| AI 해석 | OpenRouter API + DeepSeek V3.2 |
| 백엔드/DB | bkend.ai BaaS (서버 코드 0줄) |
| 이미지 생성 | html2canvas (CDN) |
| 호스팅 | Cloudflare Pages + Workers (API 키 보호) |
| CI/CD | GitHub Actions |
| 빌드 도구 | **없음** (`npx serve .`이면 끝) |

## bkend.ai — 백엔드 없이 백엔드 기능 구현

결과 공유와 통계를 위해 데이터 저장이 필요했는데, **bkend.ai BaaS**를 붙였더니 REST API 래퍼 하나(~150줄)만으로 전부 해결되어 너무 편했습니다.

```
프론트엔드 → bkend.ai REST API → MongoDB
             (테이블 생성, CRUD, 인덱스, CORS 전부 대시보드에서 클릭 몇 번)
```

별도의 백엔드 코드, ORM, 마이그레이션 스크립트가 하나도 없습니다.

### 통계 최적화 사례

| 단계 | 방식 | 문제 |
|------|------|------|
| v1 | 전체 레코드 fetch → JS forEach 집계 | 2.5MB 전송, 500건 넘으면 누락 |
| v2 | `limit=1` + `andFilters` → `pagination.total` 서버 카운팅 | 34건 개별 API 호출 |
| v3 (현재) | 서버 카운팅 + 파생값 계산 + 단일 Promise.all | 27건 → 1 round-trip, ~5KB |

핵심 최적화:
- **파생값**: `tarot = total - saju`, `female = total - male`로 API 호출 절감
- **인덱스**: `winner`, `gender`, `birth_year`, 주제별 투표 필드에 MongoDB 인덱스 설정
- **출생연도 저장**: "20대"가 아닌 `birth_year`를 저장 → 연도가 바뀌어도 정확한 연령대 계산

## 왜 Cloudflare Pages에 배포했나

| 이유 | 설명 |
|------|------|
| 무료 호스팅 | 정적 파일 무제한 대역폭, Workers 일 10만 요청 무료 |
| API 키 보안 | Pages Functions(Workers)로 OpenRouter API 키를 서버 측에서만 관리 |
| 글로벌 CDN | 별도 설정 없이 전 세계 엣지 노드에서 서빙 |
| GitHub 연동 | `main` push → 자동 배포 (GitHub Actions) |
| SSL 자동 | HTTPS 무료 자동 적용 |

## 프로젝트 구조

```
├── index.html              # SPA 메인 페이지
├── config.js               # API 프록시 URL + bkend 설정
├── css/                    # 스타일 + 애니메이션
├── js/
│   ├── saju.js             # 사주 엔진 (v2.0)
│   ├── tarot.js            # 타로 엔진 (v2.0)
│   ├── lunar.js            # 음양력 변환
│   ├── ai-interpreter.js   # AI 해석 요청 (프록시 경유)
│   ├── battle.js           # 3라운드 배틀 진행
│   ├── bkend-client.js     # bkend.ai REST 클라이언트
│   ├── stats.js            # 통계 (서버 측 카운팅)
│   ├── share.js            # SNS 공유 + 이미지 생성
│   ├── loading-messages.js # 로딩 연출 멘트
│   └── app.js              # 메인 컨트롤러
├── data/                   # 사주·타로 JSON 데이터
├── images/tarot/           # 78장 타로 카드 이미지 (Public Domain)
├── functions/api/          # Cloudflare Workers API 프록시
├── _headers                # 보안 헤더 (CSP, HSTS)
└── .github/workflows/      # GitHub Actions CI/CD
```

## 로컬 실행

```bash
npx serve .
```

AI 해석 기능은 Cloudflare Workers 프록시를 통해 동작하므로, 로컬에서 AI 기능까지 테스트하려면:

```bash
npx wrangler pages dev .
```

Cloudflare 대시보드에서 `OPENROUTER_API_KEY`와 `MODEL` 환경변수 설정이 필요합니다.

|------|------|

## 교육용 바이브코딩 사례

이 프로젝트는 **Claude Code**로 바이브코딩한 과정 자체가 교육 콘텐츠입니다. 아래는 실제 세션에서 사용된 프롬프트와 그 의도입니다.

### 도메인 지식 없이 전문 서비스 만들기

```
"/plan-plus 실제 사주와 타로를 잘 모르는 나도 잘 만들 수 있도록
너가 리드해서 디테일한 정보를 수집하고, 추측하지 말고
실제 사주와 타로 이론을 기반으로 한 서비스를 만들어보자"
```

**포인트**: "추측하지 말고 실제 이론 기반으로"라는 지시가 결과 품질을 결정합니다.

### 품질 기준을 전문가 페르소나로 설정

```
"대한민국에서 철학원만 50년 운영해 온 사주 명인이 평가해도 자신 있어?
유럽에서 30년 타로점만 봐온 집시 할머니가 평가해도 자신 있어?"
```

**포인트**: 모호한 "잘 만들어줘"가 아닌 구체적 기준을 제시하면 AI가 스스로 부족한 점을 찾아냅니다.

### AI의 비효율적 구현에 제동 걸기

```
"통계 정보만 받아와야지 왜 결과를 다 받아와?
그리고 1000건 넘으면 어떻게 할건데?"
```

```
"안되겠다. 역시 PDCA plan으로 돌려야해.
현재 통계 정보에 대해 검토하고 개선안을 가져와봐"
```

**포인트**: AI가 단순하게 구현했을 때 바로 멈추고 설계 단계로 되돌리기. 바이브코딩에서도 설계가 중요합니다.

### 데이터 모델링 방향 잡기

```
"age 20대, 30대로 하면 해가 지났을 때 그 사람들이 나이를 먹잖아.
출생 연도를 저장했다가 범위로 셀렉해"
```

**포인트**: AI에게 데이터 모델링 방향을 직접 잡아주면 훨씬 실용적인 결과가 나옵니다.

### 구체적 UX 지시

```
"응답이 오래 걸리네. 사주명인과 타로전문가가 한마디씩 던지는 느낌으로.
LLM 호출 없이 미리 100개씩 멘트 뽑아두고 랜덤 사용.
사주명인은 한자어를 섞어가며 전문성 있는 느낌으로,
타로전문가는 신비로운 말투로 기대감을 증폭시키도록"
```

**포인트**: "알아서 해줘"보다 구체적 설계를 지시하면 훨씬 좋은 결과가 나옵니다.

### 개발 여정 (커밋 30개, 세션 12개)

| 날짜 | 주요 작업 |
|------|-----------|
| 3/5 | 라이브 방송 중 서비스 초안 제작, 기본 배틀 구조 완성 |
| 3/5~7 | 타로 78장 이미지 확보, 사주/타로 엔진 전문가 수준 업그레이드 |
| 3/7~8 | OpenRouter 전환, 로딩 UX 개선, 결과 공유 기능 |
| 3/8~10 | Cloudflare 배포, GitHub Actions CI/CD, 통계 기능 |
| 3/10~11 | 만세력 정확도 교정, 7주제 선택 기능, 인생 로드맵 |
