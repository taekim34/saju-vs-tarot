# 사주 vs 타로 — 운명의 대결

> 동양의 사주(四柱)와 서양의 타로가 당신의 운세를 두고 벌이는 3라운드 AI 배틀!

## 만들게 된 계기

이 프로젝트는 [bkamp 유튜브 라이브 방송](https://www.youtube.com/live/HNjQjW37mpI?si=Z8jPH7oJJABieQOJ)에서 **바이브코딩 교육용 시연**으로 제작되었습니다. 라이브 방송 중 실시간으로 코딩을 진행하고, 방송 종료 후 완성하여 실제 배포까지 마무리한 프로젝트입니다.

"프레임워크 없이, AI와 대화하며 얼마나 빠르게 서비스를 만들 수 있는가"를 보여주기 위한 샘플입니다.

## 서비스 흐름

```
인트로 → 생년월일/성별/질문 입력 → 3라운드 배틀 → 최종 결과 → SNS 공유
```

각 라운드(연애운 → 재물운 → 종합운세)마다 사주와 타로가 각각 AI 해석을 제시하고, 사용자 투표(60%) + AI 판정(40%)으로 승자를 결정합니다.

## 사주 엔진 수준

사주 엔진(`js/saju.js`)은 단순 오락용이 아닌, **실제 전문 사주 서비스에 근접한 수준**으로 구현되어 있습니다:

| 기능 | 설명 |
|------|------|
| 사주팔자 산출 | 년주·월주·일주·시주 정확한 간지 계산 (줄리안 일수 기반) |
| 절기 정밀 계산 | 2024~2028 연도별 절기 경계 테이블 (입춘, 경칩 등 정확한 날짜) |
| 십성(十星) | 비견·겁재·식신·상관·편재·정재·편관·정관·편인·정인 — 음양 버그 수정 완료 |
| 지장간(支藏干) | 숨은 천간 추출 및 오행 가산 분석 |
| 충(冲)·합(合) | 6충 + 삼합·반합·육합 판단 및 의미 해석 |
| 대운(大運) | 10년 단위 운의 흐름 계산 (순행/역행, 현재 대운 표시) |
| 12운성 | 천간 에너지의 생로병사 순환 (장생~양) |
| 세운(歲運) | 올해 간지와 사주의 관계 분석 |
| 신강/신약 | 지장간 + 12운성 가중치 반영 정밀 판단 |
| 특수살 | 도화살·홍염살·역마살 |
| 음력 변환 | 1900~2050 음양력 변환 (외부 API 없이 자체 테이블) |

> 상용 서비스 대비 부족한 점: 절기 테이블이 2024~2028로 한정, 특수살이 3종으로 제한, 전문 역학자 감수 미진행. 오락 목적으로 충분하며, 테이블 확장과 감수를 거치면 상용 수준으로 확장 가능합니다.

## 타로 엔진 수준

타로 엔진(`js/tarot.js`)은 **Golden Dawn 전통을 반영한 전문가 수준** 분석을 제공합니다:

| 기능 | 설명 |
|------|------|
| 78장 풀 덱 | 메이저 아르카나 22장 + 마이너 아르카나 56장 (실제 카드 이미지 포함) |
| 역방향 35% | 실제 타로 리더 수준의 역방향 확률 |
| 3종 스프레드 | 쓰리 카드(과거-현재-미래), 쓰리 카드(상황-조언-결과), 호스슈(7장) |
| 원소 상성 | Elemental Dignity — 인접 카드 간 우호/적대/중립 관계 분석 |
| 수트·숫자 패턴 | 반복 수트의 원소 에너지 + 수비학적 메시지 감지 |
| 코트 카드 | 인물 에너지 분석 (시종·기사·여왕·왕) |
| 시그니피케이터 | 나이·성별·주제·사주 오행 기반 자동 배정 |
| 메이저/마이너 비율 | 운명 개입 강도 판단 |

## 기술 스택 — 의도적으로 가볍게

시연용 프로젝트이므로 **별도의 프레임워크 없이** 가장 가벼운 구성을 선택했습니다:

- **Vanilla HTML/CSS/JS** — React, Vue 없음. 빌드 도구 없음. 번들러 없음.
- **OpenRouter API** (DeepSeek V3.2) — AI 해석 생성
- **bkend.ai** — 결과 저장/공유용 BaaS (서버 코드 0줄)
- **html2canvas** — 결과 카드 이미지 생성 (CDN)

`npx serve .` 한 줄이면 로컬에서 바로 실행됩니다.

## 왜 Cloudflare Pages에 배포했나

| 이유 | 설명 |
|------|------|
| 무료 호스팅 | 정적 파일 무제한 대역폭, Workers 일 10만 요청 무료 |
| API 키 보안 | Pages Functions(Workers)로 OpenRouter API 키를 서버 측에서만 관리 |
| 글로벌 CDN | 별도 설정 없이 전 세계 엣지 노드에서 서빙 |
| GitHub 연동 | `main` push → 자동 배포 (GitHub Actions) |
| SSL 자동 | HTTPS 무료 자동 적용 |

빌드 도구가 없는 Vanilla JS 프로젝트에 가장 적합한 선택입니다. Vercel/Netlify도 가능하지만, Workers Functions와 무료 한도 면에서 Cloudflare가 이 프로젝트에 최적입니다.

## 프로젝트 구조

```
├── index.html              # SPA 메인 페이지
├── config.js               # API 프록시 URL + bkend 설정
├── css/                    # 스타일 + 애니메이션
├── js/
│   ├── saju.js             # 사주 엔진 (v2.0)
│   ├── tarot.js            # 타로 엔진 (v2.0)
│   ├── lunar.js            # 음양력 변환
│   ├── ai-interpreter.js   # AI 해석 요청 (프록시 경유)
│   ├── battle.js           # 3라운드 배틀 진행
│   ├── bkend-client.js     # 결과 저장/공유
│   ├── share.js            # SNS 공유 + 이미지 생성
│   └── app.js              # 메인 컨트롤러
├── data/                   # 사주·타로 JSON 데이터
├── images/tarot/           # 78장 타로 카드 이미지
├── functions/api/          # Cloudflare Workers API 프록시
├── _headers                # 보안 헤더 (CSP, HSTS)
└── .github/workflows/      # GitHub Actions CI/CD
```

## 로컬 실행

```bash
npx serve .
```

AI 해석 기능은 Cloudflare Workers 프록시를 통해 동작하므로, 로컬에서 AI 기능까지 테스트하려면:

```bash
npx wrangler pages dev .
```

Cloudflare 대시보드에서 `OPENROUTER_API_KEY`와 `MODEL` 환경변수 설정이 필요합니다.

## 세션당 API 호출

| 호출 | 내용 |
|------|------|
| 6회 | 사주 해석 + 타로 해석 × 3라운드 |
